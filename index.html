DNF & RPM Package Management

Installing, updating, removing, verifying, and rolling back software packages on Red Hat Enterprise Linux using DNF and RPM.

What is a Package Manager?

Package managers install, update, and remove software safely. DNF and RPM together provide a complete package management solution in RHEL.

  • DNF ‚Äì High-level tool with dependency resolution
  • RPM ‚Äì Low-level tool for queries and validation

‚¨á Installing a Package with DNF

dnf -y install wireshark

-y Automatically answers yes to prompts (use carefully in production).

Verify installation

dnf list wireshark

Updating Packages

dnf list updates
dnf -y update git
dnf update

Note: upgrade may remove obsolete packages.

‚ùå Removing Packages

dnf -y remove httpd
dnf list httpd

üïò DNF History & Rollback

dnf history
dnf -y history rollback 5

Rolls the system back to the state before transaction ID 5.

RPM Queries

rpm -ql wireshark
rpm -q --scripts httpd
rpm -q --changelog wireshark

RPM Verification

rm -f /usr/bin/wireshark
rpm -V wireshark
dnf -y reinstall wireshark

RPM Verify Attributes

SymbolDescription

Key Takeaways

  • DNF manages packages and dependencies
  • RPM validates and audits installed files
  • dnf reinstall restores missing files
  • rpm -V detects file changes
